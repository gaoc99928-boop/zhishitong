1. 产品概述

智视通是一款面向城市路口场景的边缘AI图像增强与车辆识别系统。系统部署在路口机柜内的 NVIDIA Jetson Orin Nano 边缘计算设备上，对现有监控摄像头采集的低质量图像（雨雾、夜间、逆光场景）进行实时增强与车牌识别，结构化数据直接上报交警指挥平台。

产品定位

不更换现有摄像头的"轻升级"方案 —— 通过边缘AI算法提升既有硬件在恶劣天气下的可用性，降低交管部门的设备更新成本。

4. 功能详解

4.1 图像处理流程

系统采用四阶段流水线处理每一张输入图像：

  1. 图像加载：支持拖拽上传或点击选择，自动校验格式（JPG/PNG/BMP）与大小（≤10MB）
  2. 去噪增强：对ROI区域应用 BM3D 传统去噪 + 轻量GAN深度学习增强，提升车牌区域清晰度
  3. 车辆识别：YOLOv8-Nano 检测车辆位置 → LPRNet 识别车牌字符 → ResNet18 分类车型
  4. 结果输出：生成结构化数据（车牌、车型、置信度、处理耗时），支持一键保存

4.2 设备管理

系统支持管理以下路口设备：

  • 电警相机：抓拍闯红灯、压线等违章行为，400万像素
  • 卡口相机：抓拍过车、识别车牌，400万像素
  • 全景球机：全景监控与事件复核，200万像素
  • Jetson Orin Nano：边缘计算核心，处理所有推理任务

4.3 系统设置

在「系统设置」页面可调整：

  • 去噪强度（0-100）
  • 对比度增强幅度（0-100）
  • 识别置信度阈值（50-99%，默认85%）
  • 数据上报间隔（1-60秒）
  • 交警平台API对接地址

6. 部署指南

6.1 环境准备

# 1. 刷写 JetPack 6.0 到 Jetson Orin Nano
# 2. 安装依赖
sudo apt update
sudo apt install -y python3-pip libgstreamer1.0-dev

# 3. 安装 DeepStream
sudo apt install -y deepstream-6.3

# 4. 验证 TensorRT
/usr/src/tensorrt/bin/trtexec --version

6.2 模型部署

# 转换 YOLOv8-Nano 为 TensorRT Engine
python export.py --weights yolov8n.pt --imgsz 640 --batch 1
trtexec --onnx=yolov8n.onnx --saveEngine=yolov8n.engine --fp16

# 转换 LPRNet 为 TensorRT Engine
python export_lprnet.py --weights lprnet.pt
trtexec --onnx=lprnet.onnx --saveEngine=lprnet.engine --fp16

# 放置模型到运行目录
mkdir -p /opt/zhishitong/models
cp *.engine /opt/zhishitong/models/

6.3 启动服务

cd /opt/zhishitong
python3 app.py --config config.json

# 或使用 systemd 开机自启
sudo systemctl enable zhishitong
sudo systemctl start zhishitong

7. 常见问题

Q: 系统支持哪些摄像头品牌？

A: 理论上支持所有符合 ONVIF 标准或提供 RTSP 流的海康、大华、宇视等主流品牌网络摄像头。

Q: 无网络时数据会丢失吗？

A: 不会。系统内置 SQLite 本地数据库，可缓存最近7天的识别记录。网络恢复后自动批量同步至平台。

Q: 模型可以在线更新吗？

A: 支持。云端训练新版本模型后，通过 MQTT 通道推送至边缘端，支持热切换（零停机更新）。

Q: 识别准确率受哪些因素影响？

A: 主要因素：摄像头分辨率、安装角度、车牌污损程度、天气光照条件。雨雾场景下建议开启图像增强功能。

Q: 系统功耗和散热要求？

A: Jetson Orin Nano 典型功耗15W，满负载约25W。户外部署需配置IP55防护机柜，夏季建议加装散热风扇。
